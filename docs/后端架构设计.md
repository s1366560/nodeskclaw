# 后端架构设计

## 技术栈

- Python 3.12 + FastAPI + SQLAlchemy（async） + PostgreSQL（火山云 RDS）
- 包管理：uv
- 部署：K8s（火山云 VKE）

## 核心数据模型

### workspace_tasks（工作区任务）

| 列名 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| workspace_id | VARCHAR(36) FK | 所属工作区 |
| title | VARCHAR(256) | 任务标题 |
| description | TEXT | 任务描述 |
| status | VARCHAR(20) | pending / in_progress / done / blocked / archived |
| priority | VARCHAR(16) | low / medium / high / urgent |
| assignee_instance_id | VARCHAR(36) FK | 负责 Agent 实例 |
| created_by_instance_id | VARCHAR(36) FK | 创建者 Agent 实例 |
| estimated_value | DOUBLE PRECISION | 预估价值（元） |
| actual_value | DOUBLE PRECISION | 实际价值（元） |
| token_cost | INTEGER | 消耗 token 数 |
| blocker_reason | TEXT | 阻塞原因 |
| completed_at | TIMESTAMPTZ | 完成时间 |
| archived_at | TIMESTAMPTZ | 归档时间 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |
| deleted_at | TIMESTAMPTZ | 软删除时间 |

**状态流转**：
- pending -> in_progress（Agent 接取）
- in_progress -> done / blocked
- blocked -> pending / in_progress
- done -> archived（人类管理者手动归档）
- 任何状态 -> archived

### workspace_objectives（工作区目标）

| 列名 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| workspace_id | VARCHAR(36) FK | 所属工作区 |
| title | VARCHAR(256) | 目标标题 |
| description | TEXT | 目标描述 |
| progress | DOUBLE PRECISION | 进度 0.0 ~ 1.0 |
| obj_type | VARCHAR(20) | objective（O） / key_result（KR） |
| parent_id | VARCHAR(36) FK | 父级 Objective ID（KR -> O 关联） |
| created_by | VARCHAR(36) FK | 创建者用户 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |
| deleted_at | TIMESTAMPTZ | 软删除时间 |

**层级关系**：
- `obj_type=objective`：顶层目标（O），无 parent_id
- `obj_type=key_result`：关键结果（KR），parent_id 指向所属 O

### API 路由

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /workspaces/{ws}/blackboard | 获取黑板（含 tasks + objectives） |
| GET | /workspaces/{ws}/blackboard/tasks | 任务列表（支持 ?status= 筛选） |
| POST | /workspaces/{ws}/blackboard/tasks | 创建任务 |
| PUT | /workspaces/{ws}/blackboard/tasks/{id} | 更新任务 |
| POST | /workspaces/{ws}/blackboard/tasks/{id}/archive | 归档任务 |
| GET | /workspaces/{ws}/blackboard/objectives | 目标列表 |
| POST | /workspaces/{ws}/blackboard/objectives | 创建目标 |
| PUT | /workspaces/{ws}/blackboard/objectives/{id} | 更新目标 |
| GET | /workspaces/{ws}/performance | 团队/个人效能数据 |
| POST | /workspaces/{ws}/performance/collect | 聚合 per-agent 效能统计 |
| POST | /workspaces/{ws}/performance/attribute-tokens | LLM token 归因到任务 |
| GET | /orgs/{org_id}/akr-summary | 组织级 AKR 汇总视图（EE: `akr_management`，需组织管理员） |

### 持续工作循环机制

- **定时保底**：`WorkspaceSchedule` cron 定时器，每 4 小时发送"任务巡检"系统消息给 Agent
- **事件即时**：任务状态变更时 SSE 广播 `task:status_changed`，后端同时发送系统消息提醒 Agent
- **人类手动**：管理者在黑板创建/分配任务
